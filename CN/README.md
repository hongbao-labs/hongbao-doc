# 什么是 Hongbao

> [English](../EN/README.md) · **中文**

**Hongbao**，即“红包”的拼音。

在中国传统文化中，红包是过年、婚嫁、生辰等场合里，长辈或亲友相互赠送的一只红色封套。它承载的不只是金钱，更是情感与信任。**递出去那一刻，钱就属于收红包的人，不需要任何第三方批准。** 这就是“送钱”最朴素、也最优雅的方式：一个简单的动作，就完成了所有权转移。

Hongbao 把这套体验搬到了链上。实体卡就是一张“数字红包”：卡内安全芯片在出厂时随机生成 secp256k1 私钥，并被物理永久封存。对应的链上地址持有你存入的资产；一旦把卡递给他人，资产的控制权就会完整且不可逆地转移给持卡人。**持卡即所有，签名即领取。** 这套机制不依赖任何中心化账户、激活码或冻结权限，而是基于密码学和公开合约运行。

---

## Hongbao 是什么

Hongbao 是一种**链上资产分发协议**。从外观上，它像一张实体礼品卡；从底层逻辑看，它是一种新的资产分发范式。

它适合把资产以“实物卡 + 物理转移”的方式，分发给真实用户、KOL、社区成员或线下参与者。

| 对比对象 | 我们不是 | 我们是 |
|---|---|---|
| 硬件钱包 | 面向最终用户、长期管理私钥的多次签名设备 | 面向资产分发方的一次性签名设备——卡递到收件人手里，完成所有权转移后就会作废 |
| 传统加密礼品卡 | 平台先托管资金、可冻结、可作废 | **资产锁在公开合约里**，无 owner、无 admin、不可暂停 |
| 传统空投 | 大量份额被女巫薅走、形成抛压、难以追回 | 通过实体卡进行物理筛选，强制完成钱包注册，未领取资产仍可赎回 |

---

## 它为什么与众不同

Hongbao 的核心承诺很简单：

- **持卡即所有**：私钥在芯片内由硬件 TRNG 随机生成，并被物理永久封存。卡递到谁手里，资产就属于谁。
- **签名即领取**：收款地址写入签名内容，任何中间人都无法重定向。卡只能签一次，签完后即物理作废。
- **零信任设计**：合约无 owner、无 admin、无升级。Hongbao 不接触任何资产，也不持有任何私钥。

> 协议本身在公链上自洽运行，BLE 接口、CLI 工具和合约都已公开，日常体验由官方 Web / App 提供。

---

## 不只是分发，也能做任务激励

除了“递出去就到账”之外，Hongbao 的卡还可以成为任务卡：资产拆成基础份额和任务份额。用户扫码后先领取基础份额，同时锁定收款地址；之后每完成一个任务，就能解锁对应的任务份额。

这让它不仅适合空投，也适合线下活动、会议派发、KOL 礼物、社区任务和用户增长。

---

## 它由哪些部分组成

Hongbao 由四部分协同工作：

| 组件 | 角色 | 形态 |
|---|---|---|
| **硬件签名设备**（红包卡） | 一次性 secp256k1 签名设备 | 由 Hongbao 设计、生产、销售 |
| **客户端**（Android 已上线；Web 与 iOS 即将推出） | 引导持卡人完成扫码、蓝牙连接、地址输入与签名授权 | 由 Hongbao 官方提供 |
| **标准合约** | 链上资产托管层，校验签名后放款，过期后可赎回 | 开源，可部署 / fork / 扩展 |
| **Relayer** | 代付 gas，提交领取交易 | 非信任角色，任何 EOA 都可担任 |

**链上部分完全开放**——钱包、交易所、Dapp 和项目方都可以基于这套规范集成或扩展。

---

## 它是如何工作的

每张卡都内置一颗安全芯片，在出厂时由芯片自身随机生成 secp256k1 私钥，并被物理永久封存。私钥对应的链上地址会被印成二维码，刻在卡面上。

发卡方把资产锁定在这个地址上，再把实体卡发给收件人。收件人扫码、连接蓝牙、输入收款地址、按下卡上的按键授权签名，资产就会进入指定的钱包。卡片只能签一次，签完后即物理作废。

任务卡则多了一层逻辑：同一次签名既可领取基础份额，同时也会把收款地址锁定在卡上。之后用户每完成一个任务，便可解锁对应份额，资金自动流向已锁定的地址。

---

## 适配多链，面向多种资产

Hongbao 的硬件层基于 secp256k1，这是一条加密世界事实上的通用曲线，几乎覆盖所有主流公链：

| 链 | 兼容方式 |
|---|---|
| **Bitcoin**、**Ethereum 及全部 EVM 链**、**Tron**、**Cosmos / Tendermint** 系、**Litecoin / Dogecoin / BCH** | 原生 secp256k1 账户曲线 |
| **Solana** | 原生 `secp256k1 program` precompile + `secp256k1_recover` syscall |
| **Sui** | 原生支持 secp256k1 账户曲线 |
| **Aptos** | 原生支持 secp256k1 ECDSA 交易认证器 |

当前已发布：EVM 全系（ERC20 + ERC721 标准合约）。按客户需求开放：Bitcoin、Solana、Sui、Aptos、Cosmos 等生态。

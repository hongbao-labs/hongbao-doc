## 协议概述

**Hongbao**，即"红包"的拼音。Hongbao 协议把红包范式搬到了链上：一张实体卡就是一只红包——卡内的安全芯片在出厂时随机生成 secp256k1 私钥并物理永封。递出这张卡，资产的控制权就完整地、不可逆地转移给了拿到卡的人。

**协议由四个组件协同构成：**

| 组件 | 角色 | 形态 |
|---|---|---|
| **硬件签名设备**（红包卡） | 一次性 secp256k1 签名设备 | 由 Hongbao 设计、生产、销售 |
| **客户端**（Web / Android / iOS） | 引导持卡人完成领取 | 由 Hongbao 官方提供 |
| **标准合约** | 链上资产托管层，校验签名后放款 | 开源，任何人可部署 / fork / 扩展 |
| **Relayer** | 代付 gas 提交 withdraw 交易 | 非信任角色，任何 EOA 都可担任 |

**链上部分完全开放**——钱包、交易所、Dapp 都可以基于这套规范集成或扩展。

**支持的链：** secp256k1 是加密世界事实上的通用曲线——Bitcoin、Ethereum 及全部 EVM 链、Solana、Sui、Aptos、Tron、Cosmos 等主流公链全覆盖。当前已发布：EVM 全系。其他生态按订单扩展。

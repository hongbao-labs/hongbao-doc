# 你拿到了一张红包卡

### 扫码、按一下，几十秒到账。

这张卡里装着一份链上资产——可能是稳定币（USDT / USDC）、主流代币（ETH / BNB / SOL）、NFT，也可能是项目方自己发行的任意 ERC20 / ERC721 乃至非标准代币。**具体是什么、多少数量，由发卡方决定，App 连接卡片后会显示给你看。**

它**不是兑换券、不是积分、不是凭证**——卡片对应的就是链上原生资产本身。扫一下二维码、按一下卡上的按键，几十秒就到你指定的钱包。整个领取过程不需要付 gas，也不需要懂区块链——详见 [领取指南](guide.md)。

---

## 它是什么

物理上，它和银行卡差不多大、比信用卡略厚，里面有：

- 一颗加密芯片（藏私钥的地方）
- 一颗按键
- 两颗指示灯（红、绿）
- 一颗一次性电池
- 一个二维码（印在卡面）

逻辑上，它是一张**一次性的链上资产凭证**——像现金一样，**递到谁手里，资产就属于谁**：

- **里面有一把私钥**，出厂时由芯片自身随机生成、物理永久封存。我们、发卡方、制卡厂、连你自己都没办法导出它。
- **私钥对应链上一笔锁定资产**——发卡方在你拿到卡之前已经把钱锁好。
- **卡片只能签一次**——你按按键授权之后，资产转入你指定的钱包，然后卡片物理作废。这是设计承诺，不是技术限制：每张卡保证一次确定的所有权转移，**不可挪用、不可滥用、不可双花**。

## 你需要准备什么

什么都不用。打开 Hongbao App（[TBD: 下载链接]），跟着引导走就行。

**没有任何加密钱包或交易所账号？也没关系。** App 会带你去主流加密钱包或交易所完成注册，拿到一个收款地址再回到 Hongbao 继续领取。

> 钱包私钥 / 账号始终由对应的钱包 / 交易所方掌管，**Hongbao 自己不创建、不托管、不接触你的私钥**。我们只做注册导航和派发流程。

## 为什么是安全的

四道独立的关，每一道都足以单独阻止资产丢失：

1. **私钥永远不出芯片**——从生产那天起就被封在硬件里。我们没有备份，也没办法找回——这是设计上的取舍。
2. **必须按物理按键**——蓝牙劫持没有后续：没人按按键就没有签名。
3. **收款地址写进签名里**——你输入的地址是签名内容的一部分，任何中间人想改都会让签名失效，链上合约直接拒收。
4. **卡片只能签一次**——领过的卡变成纪念品。即使有人偷走它，里面也没东西可拿。

托管资产的智能合约本身没有 owner、没有暂停按钮、没有升级机制——发卡方反悔也拿不回（除非到了过期时间）；任何中心化服务出现波动时，**你的资产在合约层永远是你的**。这部分代码完全公开，可以独立审计。

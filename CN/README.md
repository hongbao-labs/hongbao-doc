# Hongbao 协议

### Physical Bearer Token —— 链上资产的现实映射。

Hongbao 协议把"送加密货币"压缩成一个物理动作：递出一张卡。

协议的所有信任假设都收敛到密码学和公开代码——发卡方锁好资产之后，没有任何一方（包括协议作者）能改变这份资产的归属。**持卡即所有，签名即领取。**

---

## 协议组成

Hongbao 不是一台硬件，而是一套协议栈。完整协议由四个独立组件构成：

| 组件 | 角色 |
|---|---|
| **硬件签名设备**（红包卡） | 一次性 secp256k1 签名设备。私钥在芯片内自身随机生成，物理永封 |
| **标准合约** | 链上资产托管层。绑定卡片地址，校验签名后放款；过期由发卡方赎回 |
| **客户端**（Web / Android / iOS） | 引导持卡人完成扫码、蓝牙连接、收款地址输入、签名授权 |
| **Relayer** | 代付 gas 提交 withdraw 交易（任意 EOA 都可担任，非信任组件） |

四个组件**全部公开可替换**：发卡方可以套自己品牌的 App、自托管 Relayer，甚至 fork 标准合约扩展业务逻辑。我们提供完整参考实现，但不锁死任何一环。

## 怎么做到的

每张卡内置一颗 tamper-resistant 安全芯片，出厂时由芯片自身随机生成 secp256k1 私钥，物理永久封存。**包括我们在内，没有任何一方能读出这把私钥。** 私钥对应的链上地址会被印成二维码刻在卡面上。

发卡方把资产锁定到这个地址（通过协议合约托管），把实体卡寄给收件人。收件人扫码连接蓝牙、输入收款地址、按下卡上按键授权签名，资产就转入收件人指定的钱包。卡片只能签一次，签完物理作废。

中间任何环节——蓝牙链路、网络通道、代付 gas 的 relayer——都**无法**修改收款地址、双花资产、或在过期前侵占资产。这些约束写在密码学和公开合约里，不是任何一方的口头承诺。

## 一张卡，多链通吃

Hongbao 的硬件层基于 secp256k1——加密世界事实上的通用曲线。这把所有主流公链都囊括了进来：

| 链 | 兼容方式 |
|---|---|
| **Bitcoin**（含 Taproot）、**Ethereum 及全部 EVM 链**、**Tron**、**Cosmos / Tendermint** 系、**Litecoin / Dogecoin / BCH** | 原生 secp256k1 账户曲线 |
| **Solana** | 原生 `secp256k1 program` precompile + `secp256k1_recover` syscall |
| **Sui** | 原生支持 secp256k1 账户曲线（与 Ed25519 并列） |
| **Aptos** | AIP-49 起原生支持 secp256k1 ECDSA 交易认证器 + stdlib 验签模块 |

卡片本身是 chain-agnostic 的——它对 32 字节摘要做 ECDSA 签名，不关心摘要代表哪条链上的什么交易。要支持新链，只需要在那条链上部署一份对应的"锁仓-签名验证-放款"合约。卡可以原样使用。

**当前已发布**：EVM 全系（ERC20 + ERC721 标准合约）。
**按客户需求开放**：Bitcoin、Solana、Sui、Aptos、Cosmos 等其他生态。

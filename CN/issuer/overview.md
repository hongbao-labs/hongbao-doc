# Hongbao：链上资产实体分发协议

### 我们不接触你的资产、不持有任何私钥。看起来像礼品卡——底层是一种新的链上资产分发范式。

你想给一群人送加密货币——社区奖励、活动空投、KOL 礼品、会议派发、节日福利。Hongbao 把这件事变成"递出一张卡"那么简单：项目方把资产锁进公开合约，我们把卡寄给收件人，收件人扫码、按键、资产到账。**整个流程我们公司不接触一分钱**。

## 我们最重要的承诺：不碰资金

Hongbao 是加密世界第一个**真正不碰资金**的资产分发协议：

- **不持私钥**——卡内的 secp256k1 私钥由芯片内 TRNG 自生成、物理永封。我们公司、制卡厂、发卡方都没有办法读出它
- **不托管资产**——你存入合约的资产由你自己锁定，合约**无 owner、无 admin、无 pause、无升级**。我们部署完合约即退出资金链路
- **Relayer 是 optional**——`withdraw` 没有 `msg.sender` 限制，任何 EOA 都能提交。我们的默认 Relayer 只是便利工具，不是必经路径

这不是营销 claim，是**合约写死的数学保证**。结果是：**卡的领取能力由协议本身保证，独立于任何中心化服务的运营状态**——合约部署在公链上，BLE 协议和 CLI 工具完全开源，任何 EOA 都能作 Relayer。日常体验由我们的官方 Web/App 一站式把关；遇到任何问题，技术支持团队随时介入。

## 我们和最容易混淆的两类产品本质不同

### 不是硬件钱包

| 维度 | 传统硬件钱包 | Hongbao |
|---|---|---|
| **服务对象** | 资产最终持有者（个人） | 资产分发方（项目方 / 交易所 / 企业） |
| **价值主张** | 长期安全存储、多次签名 | 一次性可信分发 |
| **使用周期** | 多年 / 数十年 | 一次性，签完作废 |
| **商业模式** | 卖硬件给个人（服务溢价） | 卖分发服务给 B 端 |
| **跟用户关系** | 持续关系（长期钱包） | 一次性触达（收件人即用户） |

**核心差别**：硬件钱包是 product category（用户买回去自己用），Hongbao 是协议组件（项目方买给用户、用一次就完）。**"一次性"在 Hongbao 不是技术限制，是设计承诺**——保证每张卡只代表一次确定的所有权转移，不可挪用、不可滥用、不可双花。

### 不是传统加密礼品卡

| 维度 | 传统加密礼品卡 | Hongbao |
|---|---|---|
| **资金托管** | 平台托管用户资产 | **不接触资金**，项目方自锁合约 |
| **支持资产** | 通常仅单一币种 | 任意 secp256k1 链 / 任意 ERC20 / NFT |
| **未领取资产** | 平台留存 | 时间锁到期由发卡方一键赎回 |
| **安全模型** | 中心化（平台单点风险） | 去中心化（链上可验证） |
| **硬件保护** | 无（纸卡 / 电子码 / 涂层）| 安全芯片 + 物理按键 + 一次性签名 |

**核心差别**：传统加密礼品卡的资金流是 "用户付款 → 平台托管 → 平台发放"——平台全程控制资金。Hongbao 的资金流是 "项目方自锁合约 → 用户硬件提取"——我们全程不接触资金。**这不是稍微好一点，是架构层面的代际差异**。

## 与传统礼品卡的核心差异

如果你之前用过传统礼品卡作为派发工具，下面这张表是 Hongbao 和它们的对照：

| 风险 | 传统礼品卡 | Hongbao |
|---|---|---|
| **发卡平台运营波动** | 沉淀资金归属不清 / 余额可能作废 | 资产锁在公开合约里，与任何中心化服务的运营状态无关——链还在、卡还在就能领 |
| **卡密数据库泄漏** | 整批卡可能作废 / 大规模盗刷 | 没有"卡密数据库"——每张卡的价值由芯片内私钥决定，攻不破单一目标就攻不破任何卡 |
| **卡号 / CVC 被偷拍** | 涂层易刮、卡面可扫拍、信息即价值 | 卡上的二维码本身**没有**兑付能力——必须有物理卡 + 按物理按键签名才能解锁 |
| **激活窗口攻击** | 激活前后被盗刷（典型欺诈手法） | 没有激活流程。deposit 完成那一刻卡就处于可领状态 |
| **平台规则变更** | 发卡方可能调整兑付规则、冻结余额 | 合约去中心化、无 admin、不可暂停 |
| **信任假设** | 必须信任发卡公司维持兑付能力、不挪用预付金 | 只需信任卡片硬件 + 公开合约代码 |

核心差异是**信任模型**：传统礼品卡是"信任发卡公司"；Hongbao 是"信任密码学和公开代码"。

## 对你的实际价值

### 1. 真实用户筛选 + 新用户 onboarding

**对发币 / 空投场景**：链上空投的真实用户留存通常只在 5-15%，且大量份额被女巫地址领走形成抛压。物理卡片把领取门槛抬到"必须有卡 + 物理按键"——批量脚本无法绕过，代币精确到达真实人手里。

**对新用户进圈场景**：没有钱包、没有交易所账号的用户也能领。我们的 App 内置主流加密钱包和交易所的注册导航，新用户完成注册后回到领取流程，资产直接到账——你触达的不只是已有钱包的链上活跃用户，也包括过去因 onboarding 漏斗失联的潜在新用户。

### 2. 风险锁死的设计

下单后，你最坏情况的损失**在合约层就被锁死**：

- **正常情况** —— 高领取率：真实用户完成钱包注册，代币精确到达活跃地址
- **最坏情况**（极端 0% 领取率）—— 你只损失硬件费 + 服务费（典型订单 $5-15k）；底层资产通过合约 `withdrawExpired` 全数赎回回你的钱包

这不是我们的承诺，是合约写死的数学保证——签合同前你可以自己审代码验证。

对比传统空投：大量份额一去不回。对比传统礼品卡：沉淀资金归属不清。**Hongbao 把你最坏情况下的损失锁死在卡片成本，底层资产永远在你掌控里。**

### 3. 单位经济学参考

下面的对照基于行业公开基准——Hongbao 的实际表现会随着每一次合作积累的实测数据持续更新。**领取数据通过 Web Dapp 后台实时透明，链上事件可独立验证。**

| 指标 | 行业基准 | Hongbao 目标范围 |

| 单个真实用户综合成本（CAC） | $40-100 | 按预期领取率折算 < $20 |
| 真实用户 30 天留存 | 5-15% | 物理实物 + 强制钱包注册，预期显著提升 |

## 工作流

```
1. 下单（数量 / 目标链 / 资产 / 卡面定制方案）
2. 我们寄出实体卡 + 一份 JSON（含每张卡的链上地址 + 批次元数据）
3. 项目方收货验证（抽样或全量比对地址）
4. 登录 Hongbao Web Dapp，一键锁定资产到这批卡
5. 派发实体卡给收件人
6. 持卡人扫码 → 按键 → 资产到账 → 卡片作废
7. （可选）过期未领的资产在 Web Dapp 一键赎回
```

项目方真正需要操作的只有三步：**第 4 步一键锁定** + **第 5 步派发实体卡给收件人** + **第 7 步一键赎回**。Factory / Pool 部署、approve、batchDeposit、withdrawExpired 等链上动作全部由 Hongbao Web Dapp 集成在按钮背后，不需要写脚本、不需要直接调合约。详细流程见 [端到端指南](guide.md)；想直接走合约层 / 自建客户端的开发者参考 [dev/](../dev/)。

## 你能做的 / 不能做的

**你能做的**：

- 给任意 secp256k1 链上的资产发卡（当前 EVM 首发，含 ERC20 + ERC721；其他生态按订单扩展，详见 [README](../README.md)）
- 设置过期时间（最少 30 天）
- 过期未领时一键赎回全部
- 卡面联名定制（保留 Hongbao logo，其余区域开放给你的视觉设计，详见 [customization.md](customization.md)）
- 用 Hongbao 默认 Relayer 代付 gas，或自托管 Relayer / 让用户自付（持卡人入口统一是 Hongbao 官方 App / Web）

**你不能做的**（设计上的硬约束，也是收件人愿意把它当作"真礼物"的原因）：

- 改变某张卡的收款地址（持卡人完全控制）
- 在过期前撤回任何资产（持卡人对未过期的卡有完整所有权）
- 冻结 / 暂停某张卡（合约不存在 owner / admin）
- 提取手续费（合约层面无费用）

## 商业模式

Hongbao 链上零费用——**合约层零手续费、Relayer 零手续费**。你的成本只有硬件 + 资产本身，链上没有任何隐藏抽成。这是刻意如此：让 Hongbao 在合约 / 资金面对发卡方完全透明，与发卡方的合规、税务、风控解耦。

我们的收入全部在协议层之外，由三条线构成：

| 收入线 | 说明 |
|---|---|
| **硬件销售** | 卡片售价。B 端批量定价 + C 端零售 |
| **项目方服务费** | 卡面联名定制、新链 / 新资产生态扩展、集成支持等定制项目 |
| **App 端生态合作** | 持卡人在 App 内对接钱包、交易所、Dapp 时产生的引流分成（与发卡方的资产托管无任何关系） |

现阶段主要客户是 **Web3 项目方 / 公链生态 / 交易所 / 钱包**——典型场景包括 KOL 礼品、早鸟空投替代、社区 OG 奖励、会议派发、稳定币发行方的新用户 onboarding 工具。我们的协议同样适用于更广阔的零售场景，未来与合作伙伴一起拓展。批量采购与定制详见 [contact.md](../contact.md)。

## 安全保证

- **私钥从未存在 Hongbao 任何系统中**——每张卡的私钥由芯片自身在出厂时生成、物理永封
- **Hongbao 公司不持有任何资产**——deposit 后资产在你部署的合约里，与我们的运营状态完全解耦
- **去信任合约**——无 owner、无 pause、无升级、无后门。合约部署在公链上，BLE 接口和 CLI 工具完全开源——卡的领取能力**由协议本身保证**

团队成员均为安全背景（含合约审计员）；代码完全公开，欢迎独立审计或自审。具体合作条款、责任划分以及双方义务详见正式合同。

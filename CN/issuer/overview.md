# 去中心化礼品卡

### 资产链上自托管，无需信任发卡公司，无激活流程，无中心化数据库可被攻击。

你想给一群人送加密货币——社区奖励、活动空投、员工年终、客户礼品、节日发放。Hongbao 协议把这件事变成"递出一张卡"那么简单，但底层模型和你熟悉的 Visa Prepaid / Apple Gift Card / 京东 E 卡完全不同。

---

## 与传统礼品卡的差异

传统礼品卡的结构：项目方 → 把资金交给发卡公司 → 发卡公司中心化保管 → 发卡公司印卡 → 用户激活 / 兑付。

| 风险 | 传统礼品卡 | Hongbao |
|---|---|---|
| **发卡公司倒闭 / 服务中断** | 沉淀资金消失 / 余额作废 | 资产锁在公开合约里，与任何中心化服务的运营状态无关——链还在、卡还在就能领 |
| **发卡公司被黑** | 卡密数据库泄漏 → 全部作废 / 大规模盗刷 | 没有"卡密数据库"——每张卡的价值由芯片内私钥决定，攻不破单一目标就攻不破任何卡 |
| **卡号 / CVC 被偷拍** | 涂层易刮、卡面可扫拍、信息即价值 | 卡上的二维码本身**没有**兑付能力——必须有物理卡 + 按物理按键签名才能解锁 |
| **激活窗口攻击** | 激活前后被盗刷（典型欺诈手法） | 没有激活流程。deposit 完成那一刻卡就处于可领状态 |
| **监管 / 合规冻结** | 发卡公司被监管 → 资金冻结 → 卡作废 | 合约去中心化、无 admin、不可暂停。与发卡公司监管状态无关 |
| **信任假设** | 必须信任发卡公司维持兑付能力、不挪用预付金 | 只需信任卡片硬件 + 公开合约代码 |

核心差异是**信任模型**：传统礼品卡是"信任发卡公司"；Hongbao 是"信任密码学和公开代码"。

Hongbao 是硬件供应商 + 协议运营方，不是托管方——deposit 之后我们对你的资产**没有任何接触面**。这是产品的核心设计，也是合规的最低摩擦点。

## 工作流

```
1. 下单（数量 / 目标链 / 资产 / 卡面定制方案）
2. 我们寄出实体卡 + 一份 JSON（含每张卡的链上地址 + 批次元数据）
3. 项目方收货验证（抽样或全量比对地址）
4. 登录 Hongbao Web Dapp，一键锁定资产到这批卡
5. 派发实体卡给收件人
6. 持卡人扫码 → 按键 → 资产到账 → 卡片作废
7. （可选）过期未领的资产在 Web Dapp 一键赎回
```

项目方真正需要操作的只有两步：**第 4 步一键锁定** + **第 7 步一键赎回**。Factory / Pool 部署、approve、batchDeposit、withdrawExpired 等链上动作全部由 Hongbao Web Dapp 集成在按钮背后，不需要写脚本、不需要直接调合约。详细流程见 [端到端指南](guide.md)；想直接走合约层 / 自建客户端的开发者参考 [dev/](../dev/)。

## 你能做的 / 不能做的

**你能做的**：

- 给任意 secp256k1 链上的资产发卡（当前 EVM 首发，含 ERC20 + ERC721；其他生态按订单扩展，详见 [README](../README.md)）
- 设置过期时间（最少 30 天）
- 过期未领时一键赎回全部
- 卡面联名定制（保留 Hongbao logo，其余区域开放给你的视觉设计，详见 [customization.md](customization.md)）
- 用 Hongbao 默认 Relayer 代付 gas，或自托管 Relayer / 让用户自付（持卡人入口统一是 Hongbao 官方 App / Web）

**你不能做的**（设计上的硬约束，也是收件人愿意把它当作"真礼物"的原因）：

- 改变某张卡的收款地址（持卡人完全控制）
- 在过期前撤回任何资产（持卡人对未过期的卡有完整所有权）
- 冻结 / 暂停某张卡（合约不存在 owner / admin）
- 提取手续费（合约层面无费用）

## 商业模式

Hongbao 链上零费用——**合约层零手续费、Relayer 零手续费**。你的成本只有硬件 + 资产本身，链上没有任何隐藏抽成。这是刻意如此：让 Hongbao 在合约 / 资金面对发卡方完全透明，与发卡方的合规、税务、风控解耦。

我们的收入全部在协议层之外，由三条线构成：

| 收入线 | 说明 |
|---|---|
| **硬件销售** | 卡片售价。B 端批量定价 + C 端零售 |
| **项目方服务费** | 卡面联名定制、新链 / 新资产生态扩展、集成支持等定制项目 |
| **App 端生态合作** | 持卡人在 App 内对接钱包、交易所、Dapp 时产生的引流分成（与发卡方的资产托管无任何关系） |

主要客户是 B 端项目方 / 企业——社区空投、节日发放、员工奖金、合作礼品、白名单凭证等批量场景；同时也有少量 C 端零售（个人买来送礼）。批量采购与定制详见 [contact.md](../contact.md)。

## 安全与合规

- **私钥从未存在 Hongbao 任何系统中**——每张卡的私钥由芯片自身在出厂时生成。
- **Hongbao 公司不持有任何资产**——deposit 后资产在你部署的合约里，与我们的运营状态完全解耦。
- **去信任合约**——无 owner、无 pause、无升级、无后门。即便 Hongbao 服务中断、服务器被攻击或官方 App 下线，已发出的卡仍能正常被领。
- **合规接触面最小**——具体派发动作（你给谁发了多少）由你完成，相关税务、KYC、AML 由你承担。我们的协议设计刻意不接触这些。

团队成员均为安全背景（含合约审计员）；当前阶段未做第三方审计，代码完全公开，欢迎独立审计或自审。

# 一张实体卡 = 一笔链上资产的物理凭证

### 把加密资产精准送到真实用户手里——不持币、不托管、不被女巫薅走、不在邮寄环节出事。

Hongbao 是一种新的链上资产分发协议——表面上像礼品卡，本质是一种从未存在过的范式：**实体形态 + 链上自托管**。项目方把资产锁进公开合约，我们把卡寄给收件人，收件人扫码、按键、资产到账。整个流程我们公司部署完合约即退出资金链路，**资产由协议本身保证，独立于任何中心化服务的运营状态**。

## 你下一次派发，会面临什么？

如果你做过空投、活动派发、KOL 礼品、社区奖励——你大概率经历过下面其中之一。

### 1. 纸片卡的物理风险

把私钥或 redeem 码印在纸片上，**从打印到邮寄到签收每一个环节都是攻击面**：

- **晒单泄露**：Brazilian YouTuber 直播时意外暴露 seed phrase，数分钟内被观众取走价值约 $60K 的 MATIC + NFT ([Decrypt 报道](https://decrypt.co/154522/youtuber-loses-60k-worth-crypto-after-showing-seed-phrase-stream))
- **内部盗窃**：Home Depot 一名员工利用内部访问权限在 16 个月内盗取 8,000+ 张实体礼品卡，价值 $4M ([CBS 报道](https://www.cbsnews.com/atlanta/news/ex-home-depot-employee-sentenced-to-prison-for-stealing-over-4-million-in-gift-card-scheme-officials-say/))
- **物理 Bitcoin 教训**：DEFCON 2013 上 hacker 公开演示，hologram 揭开后可几乎无痕替换——意味着供应链任何一环都能私拷私钥再伪装成"未启封" ([CoinDesk 报道](https://coindesk.com/markets/2013/08/13/defcon-hackers-crack-physical-bitcoin-casascius-coins/amp))
- **公共场所诱饵**：Vienna / 新南威尔士警方多次警告街头分发的"免费 Bitcoin 纸钱包"——扫码即钓鱼陷阱

### 2. 沉没预算（过期未领的 breakage）

实体派发的卡片永远有相当大比例从未被领取——这部分预算就这么沉没了：

- 美国零售礼品卡每年 **$21B 永远沉没**（10-19% 不被领取）；2023→2024 unredeemed 金额涨幅 30.5% ([Capital One Shopping 调研](https://capitaloneshopping.com/research/gift-card-statistics/))
- 加密 airdrop 5-50% 未领取；MOVE 空投 5.8 亿 / 10 亿 token（**58%**）未领取 ([CoinLaw 数据](https://coinlaw.io/token-airdrop-statistics/))

发卡方真实承担了硬件 + 资产成本，**纸片卡形式下这部分钱回不来**。

### 3. 激活率灾难（新用户进圈失败）

新用户拿到纸片卡 / 兑换码的体验是：撕膜 → 看到一串字符串 → 不知道这是什么 → 下载某个钱包 → 导入私钥——对加密小白是噩梦。每掉 10 个百分点的激活率，你的有效 CAC 翻一倍以上。

- Web3 项目方烧在用户获取上的支出占企业价值 **29-95%**（Web2 仅 7-15%）—— 3-7x 的 CAC 倍数 ([Tomtunguz: Tokens as CAC](https://tomtunguz.com/tokens-as-cac/))
- DeFi 单真实用户 CAC **$85+**；空投 retained user 高达 **$500-1,000+** ([Web3 User Acquisition Trends](https://www.blockchain-ads.com/post/user-acquisition-trends-report))
- MetaMask 自承：Social Login 把 onboarding 摩擦降了 30%——意味着原始 seed-phrase 流程的流失率高得离谱

### 4. 女巫攻击（份额被机器人薅走）

链上空投的真实用户份额被 sybil 农场捕获——发卡方付了真金白银，**但代币没到真人手里**：

- Sybil 攻击者捕获了某些大型空投 **48% 的 token** ([Cointelegraph 报道](https://cointelegraph.com/news/token-airdrops-targeted-farm-accounts-sybil-attacks))
- Cookie3 调研：airdrop "eligible" 钱包里 **70% 是假的**
- 行业基准：LayerZero 单次过滤 80 万个 sybil 地址；Starknet 70 万 ([Sybil Attacks 综述](https://integral.xyz/blog/sybil-attacks))

### 5. 多链 SKU 噩梦

项目方在 Polygon + BSC + Solana 各做一次活动——纸片卡形式下至少要做 **3 种独立 SKU**（每条链私钥编码格式 / 钱包导入流程 / 兑换文档都不同）。库存复杂度上升，MOQ 分散，单价上升，前置周期延长。

## Hongbao 怎么解决

### 5 个痛点 → 5 个规避机制

| 痛点 | Hongbao 怎么避免 |
|---|---|
| **纸片卡物理风险** | 私钥**在芯片内自生成**——出厂前没有任何人/系统读到过它。卡上 QR 码只是 ETH 地址指针，**不能签名**——晒单 / 拍照 / 偷拍都不构成损失。物理按键 + 安全芯片防拆，从架构上消除供应链与中途泄露的攻击面 |
| **沉没预算** | 30 天锁定期后发卡方调用 `withdrawExpired` 把未领取部分**全数赎回回钱包**。Breakage 损失从 10-50% 降到 0 |
| **激活率灾难** | App 引导：扫码 → 完成钱包/交易所注册 → 按按键 → 资产到账。新用户**无需先懂** seed phrase / 钱包导入，激活路径平直，链上小白也能完成 |
| **女巫攻击** | 物理卡 + 物理按键不可批量。脚本和 sybil 农场过不了"必须有卡 + 长按 10 秒"这一关——代币精确到达**真实人手里** |
| **多链 SKU** | secp256k1 是几乎所有主流链共用的签名曲线（Bitcoin / 全部 EVM / Tron / Cosmos / Solana / Sui / Aptos）——**一个 SKU 适配所有链**，只需在目标链各部署一份合约 |

### 三个核心承诺

支撑上面所有规避机制的，是合约 / 硬件层的三个硬性承诺：

- **持卡即所有** —— 私钥在芯片内由硬件 TRNG 生成、物理永封，无人可读出。卡递到谁手里，资产属于谁
- **签名即领取** —— 收款地址写进签名内容，任何中间人无法重定向。卡只能签一次，签完物理作废：这是设计承诺，不是技术限制——每张卡保证**一次确定的所有权转移**，像现金一样不可挪用
- **零信任设计** —— 合约无 owner、无 admin、无升级。Hongbao 公司部署完合约即退出资金链路。卡的领取能力**由协议本身保证**，独立于任何中心化服务的运营状态

## 容易混淆的两类产品，我们和它们本质不同

### 不是硬件钱包

| 维度 | 传统硬件钱包 | Hongbao |
|---|---|---|
| **服务对象** | 资产最终持有者（个人） | 资产分发方（项目方 / 交易所 / 企业） |
| **价值主张** | 长期安全存储、多次签名 | 一次性可信分发 |
| **使用周期** | 多年 / 数十年 | 一次性，签完作废 |
| **跟用户关系** | 持续关系（长期钱包） | 一次性触达（收件人即用户） |

**核心差别**：硬件钱包是 product category（用户买回去自己用），Hongbao 是协议组件（项目方买给用户、用一次就完）。

### 不是传统加密礼品卡

| 维度 | 传统加密礼品卡 | Hongbao |
|---|---|---|
| **资金托管** | 平台托管用户资产 | **不接触资金**，项目方自锁合约 |
| **支持资产** | 通常仅单一币种 | 任意 secp256k1 链 / 任意 ERC20 / NFT |
| **未领取资产** | 平台留存 | 时间锁到期由发卡方一键赎回 |
| **安全模型** | 中心化（平台单点风险） | 去中心化（链上可验证） |
| **硬件保护** | 无（纸卡 / 电子码 / 涂层） | 安全芯片 + 物理按键 + 一次性签名 |

**核心差别**：传统加密礼品卡的资金流是 "用户付款 → 平台托管 → 平台发放"——平台全程控制资金。Hongbao 的资金流是 "项目方自锁合约 → 用户硬件提取"——我们全程不接触资金。**这不是稍微好一点，是架构层面的代际差异。**

## 工作流

```
1. 下单（数量 / 目标链 / 资产 / 卡面定制方案）
2. 我们寄出实体卡 + 一份 JSON（含每张卡的链上地址 + 批次元数据）
3. 项目方收货验证（抽样或全量比对地址）
4. 登录 Hongbao Web Dapp，一键锁定资产到这批卡
5. 派发实体卡给收件人
6. 持卡人扫码 → 按键 → 资产到账 → 卡片作废
7. （可选）过期未领的资产在 Web Dapp 一键赎回
```

项目方真正需要操作的只有三步：**第 4 步一键锁定** + **第 5 步派发实体卡** + **第 7 步一键赎回**。Factory / Pool 部署、approve、batchDeposit、withdrawExpired 等链上动作全部由 Web Dapp 集成在按钮背后，不需要写脚本、不需要直接调合约。详细流程见 [端到端指南](guide.md)；想直接走合约层 / 自建客户端的开发者参考 [dev/](../dev/)。

## 你能做的 / 不能做的

**你能做的**：

- 给任意 secp256k1 链上的资产发卡（当前 EVM 首发，含 ERC20 + ERC721；其他生态按订单扩展，详见 [README](../README.md)）
- 设置过期时间（最少 30 天）
- 过期未领时一键赎回全部
- 卡面联名定制（保留 Hongbao logo，其余区域开放给你的视觉设计，详见 [customization.md](customization.md)）
- 用 Hongbao 默认 Relayer 代付 gas，或自托管 Relayer / 让用户自付（持卡人入口统一是 Hongbao 官方 App / Web）

**你不能做的**（设计上的硬约束，也是收件人愿意把它当作"真礼物"的原因）：

- 改变某张卡的收款地址（持卡人完全控制）
- 在过期前撤回任何资产（持卡人对未过期的卡有完整所有权）
- 冻结 / 暂停某张卡（合约不存在 owner / admin）
- 提取手续费（合约层面无费用）

## 商业模式

Hongbao 链上零费用——**合约层零手续费、Relayer 零手续费**。你的成本只有硬件 + 资产本身，链上没有任何隐藏抽成。这是刻意如此：让 Hongbao 在合约 / 资金面对发卡方完全透明，与发卡方的合规、税务、风控解耦。

我们的收入全部在协议层之外：

| 收入线 | 说明 |
|---|---|
| **硬件销售** | 卡片售价。B 端批量定价 + C 端零售 |
| **项目方服务费** | 卡面联名定制、新链 / 新资产生态扩展、集成支持等定制项目 |
| **App 端生态合作** | 持卡人在 App 内对接钱包、交易所、Dapp 时产生的引流分成（与发卡方的资产托管无任何关系） |

现阶段主要客户是 **Web3 项目方 / 公链生态 / 交易所 / 钱包**——典型场景包括 KOL 礼品、早鸟空投替代、社区 OG 奖励、会议派发、稳定币发行方的新用户 onboarding 工具。我们的协议同样适用于更广阔的零售场景，未来与合作伙伴一起拓展。批量采购与定制详见 [contact.md](../contact.md)。

## 安全保证

- **私钥从未存在 Hongbao 任何系统中** —— 每张卡的私钥由芯片自身在出厂时生成、物理永封
- **Hongbao 公司不持有任何资产** —— deposit 后资产在你部署的合约里，与我们的运营状态完全解耦
- **去信任合约** —— 无 owner、无 pause、无升级、无后门。合约部署在公链上，BLE 接口和 CLI 工具完全开源——卡的领取能力**由协议本身保证**

团队成员均为安全背景（含合约审计员）；代码完全公开，欢迎独立审计或自审。具体合作条款、责任划分以及双方义务详见正式合同。

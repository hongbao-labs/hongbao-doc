# 系统架构

本节面向想要**接入或扩展** Hongbao 的开发者，介绍系统的对外接口与组件职责。

## 组件全景

```
 ┌──────────┐  BLE   ┌─────────┐ HTTPS  ┌──────────┐ on-chain ┌────────┐
 │ 红包卡   │◄──────►│   App   │◄──────►│ Relayer  │◄────────►│ Pool   │
 │（设备）  │        │         │  (可选) │ (可选)   │          │ 合约   │
 └──────────┘        └─────────┘        └──────────┘          └────────┘
   secp256k1        BLE 客户端          代付 gas             EIP-712 验证
   一次性签名        交互编排            提交 withdraw         资产托管
```

四个组件，各自独立。其中：

- **必须存在**：设备、合约
- **可替换**：App、Relayer——任何人可以自建

## 各组件能做 / 不能做

| 组件 | 知道私钥？ | 知道收款地址？ | 能改收款地址？ | 能提交链上交易？ |
|---|---|---|---|---|
| 红包卡 | ✅（永不导出）| ❌ | ❌ | ❌ |
| App | ❌ | ✅（用户输入）| 改了签名失效 | ❌ |
| Relayer | ❌ | ✅（从签名内容反推） | 改了签名失效 | ✅ |
| Pool 合约 | ❌ | ✅（从 ecrecover 反推 + signed message）| ❌ | N/A |

**关键不变量**：收款地址 `to` 是签名内容的一部分。任何环节修改 `to`，签名都会失效，合约 `ecrecover` 会拒绝。这把"中间人重定向资产"的攻击面缩小到 0。

## 信任模型

需要信任的：

1. **芯片厂商**做出的 secure element 真的是 tamper-resistant（私钥不可提取、HMAC 不可绕过）。
2. **secp256k1 + ECDSA + keccak256** 这套密码学原语是安全的（这是以太坊本身的信任假设）。
3. **持卡人**自己保管好卡片。

不需要信任的：

- 我们公司（不持有任何卡的私钥；合约无 admin）
- 发卡方（不能改你的收款地址；不能在过期前撤回）
- App（开源 + 可自建；改签名内容只会让签名失效）
- Relayer（任意 EOA 都能当 Relayer；唯一作用是付 gas + 调合约）
- BLE 链路（不加密——但传输的内容里没有秘密；签名是公开数据）

## 端到端 claim 时序

```
持卡人          App                设备              Relayer            合约
   │             │                  │                  │                 │
   │ 扫码        │                  │                  │                 │
   │────────────►│                  │                  │                 │
   │             │ 解析 deeplink    │                  │                 │
   │             │ 4字节地址前缀    │                  │                 │
   │             │                  │                  │                 │
   │             │ BLE scan + connect│                  │                 │
   │             │─────────────────►│  PENDING         │                 │
   │             │                  │  红绿交替闪      │                 │
   │ 长按 3 秒   │                  │                  │                 │
   │═══════════════════════════════►│  CONFIRMED       │                 │
   │             │                  │                  │                 │
   │             │ IDENTITY_CHALLENGE                  │                 │
   │             │─────────────────►│                  │                 │
   │             │◄─────────────────│ 签名 + identity pubkey              │
   │             │                  │                  │                 │
   │             │ 校验 pubkey == QR里的 identity      │                 │
   │             │                  │                  │                 │
   │             │ 读链上 cardTotal/cardExpire/...    │                 │
   │             │─────────────────────────────────────────────────────►│
   │             │◄─────────────────────────────────────────────────────│
   │             │                  │                  │                 │
   │ 输入 to     │                  │                  │                 │
   │────────────►│                  │                  │                 │
   │             │ digest = getWithdrawDigest(unlockAddress, to)         │
   │             │─────────────────────────────────────────────────────►│
   │             │◄─────────────────────────────────────────────────────│
   │             │                  │                  │                 │
   │             │ SIGN(digest)     │                  │                 │
   │             │─────────────────►│  WAITING_BUTTON  │                 │
   │             │                  │  红灯闪烁        │                 │
   │ 长按 10 秒  │                  │                  │                 │
   │═══════════════════════════════►│  签名            │                 │
   │             │◄─────────────────│ 64B 签名         │                 │
   │             │                  │  红+绿常亮（已用）                   │
   │             │ 推导 v（试 27/28）│                  │                 │
   │             │                  │                  │                 │
   │             │ POST {unlockAddr, to, v, r, s} ───►│                 │
   │             │                  │                  │ withdraw(...) ─►│
   │             │                  │                  │◄─── tx hash ────│
   │             │◄─────────────────────────────────── │                 │
   │             │                  │                  │                 │
   │             │ 显示成功         │                  │                 │
```

时序里有四个独立的"闸门"：

1. **物理按键 3 秒**——把 BLE 通道从 OPEN 升到 CONFIRMED
2. **身份验证**——把通道升到 IDENTIFIED（这是签名所必需）
3. **物理按键 10 秒**——授权签名
4. **链上 ecrecover**——验证签名 = unlockAddress 且对当前 (chainId, pool)

任意一个闸门失败，资产都不会动。
